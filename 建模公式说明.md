# 建模公式说明（与代码函数对应）

> **论文叙事体例**（问题背景 → 建模思路 → 公式含义、Model A–D 联调、双层规划合理性）见 **[`论文初稿.md`](./论文初稿.md)**。本文件侧重**与代码函数一一对应**的公式条目。

下文符号与实现一致；离散场景集合为 \(S\)，概率 \(\pi_s\)。CVaR 的 Rockafellar–Uryasev 线性化统一为：辅助变量 \(\zeta\) 与 \(u_s\ge 0\)，

\[
\mathrm{CVaR}_\beta(L) = \zeta + \frac{1}{1-\beta}\sum_{s\in S}\pi_s\,u_s,\qquad
u_s \ge L_s - \zeta
\]

（若 \(L_s\) 为向量最大值，则用标量 \(u_s\) 对每个分量约束 \(u_s\ge L_{s,j}-\zeta\) 再取公共 \(u_s\)，与代码一致。）

## 一、`TEAVAR_Gurobi.TEAVAR`

**决策**：每条 flow \(f\) 在其候选隧道索引 \(t\) 上的流量 \(a_{f,t}\ge 0\)；\(\alpha\ge 0\)；\(u_{s,f}\ge 0\)；\(u^{\max}_s\ge 0\)。

**记号**：\(d_f\) 为需求；\(X_{s,t}\in\{0,1\}\) 为场景 \(s\) 下隧道 \(t\) 是否可用；分配在隧道上的满足量为 \(\sum_t a_{f,t} X_{s,T(f,t)}\)（实现中以索引矩阵表达）。

**单 flow 损失（未满足比例下界）**：

\[
u_{s,f} \;\ge\; 1 - \frac{1}{d_f}\sum_{t} a_{f,t}\,X_{s,T(f,t)} \quad (\text{若 } d_f>0;\ \text{否则 } u_{s,f}=0)
\]

**场景级聚合**（`average=False` 时为最坏 flow）：

\[
u^{\max}_s + \alpha \;\ge\; u_{s,f}\quad \forall f
\]

（`average=True` 时用平均损失代替逐 \(f\) 约束。）

**链路容量**：对每条边 \(e\)，\(\sum_{f,t} a_{f,t}\,L_{t,e} \le \mathrm{cap}_e\)（\(L_{t,e}\) 为隧道–边关联。）

**目标**：

\[
\min\ \alpha + \frac{1}{1-\beta}\sum_{s}\pi_s\,u^{\max}_s
\]

## 二、`duibi.build_single_layer_model`

**决策**：\(y_{im}\in\{0,1\}\) 任务 \(i\) 是否置于节点 \(m\)；\(x^{in}_{i,m,p},x^{out}_{i,m,q}\ge 0\) 为路径流量；\(\zeta_N,\zeta_L\) 自由；\(u_s,v_s\ge 0\)。

**成本**：

\[
c_p = \sum_{i,m} y_{im}\sum_k w_{ik}\,\pi_{mk},\quad
c_b = \sum_{i,m,p} x^{in}_{i,m,p}\,|p| + \sum_{i,m,q} x^{out}_{i,m,q}\,|q|
\]

（\(|p|\) 为路径 hop 数，代码用 `len(path)`。）

**算力利用率 CVaR**（与场景容量 \(C^N_{mks}\) 对齐）：

\[
\mathrm{CVaR}^N = \zeta_N + \frac{1}{1-\beta_N}\sum_s \pi_s u_s,\qquad
u_s \ge \frac{\sum_i w_{ik} y_{im}}{C^N_{mks}} - \zeta_N\quad \forall s,m,k
\]

**链路利用率 CVaR**（边容量 \(B_e\,\sigma_{es}\)）：

\[
\mathrm{CVaR}^L = \zeta_L + \frac{1}{1-\beta_L}\sum_s \pi_s v_s,\qquad
v_s \ge \frac{\mathrm{flow}_e(x,s)}{B_e\,\sigma_{es}} - \zeta_L\quad (\sigma_{es}>0)
\]

**主要约束**：\(\sum_m y_{im}=1\)；\(\sum_p x^{in}_{i,m,p}=y_{im} b^{in}_i\)，\(\sum_q x^{out}_{i,m,q}=y_{im} b^{out}_i\)；名义算力 \(\sum_i w_{ik} y_{im}\le C^{norm}_{mk}\)。

**流锚点与候选路径**：代码用 `teavar_flow_anchors(data)` 得到 \((s_{\mathrm{src}},s_{\mathrm{dst}})\)。hub 径向时为 \((h,h)\)，候选为 \(P_{hm},P_{mh}\)（\(h=\texttt{getattr(data,'hub',0)}\)）；UMCF 开启时为 \((V_s,V_t)\)，候选为 \(P_{V_s m},P_{m V_t}\)，与 `b4_joint_data` 的 `P_cand` 扩展一致。放置与 `min_off_hub` 仍用物理 hub \(h\)。

**链路利用率指标**：`duibi_metrics` 中 max link util 等遍历 `data.E` 时跳过 UMCF 辅助边（无穷容量），避免指标被虚拟边稀释。

**目标**：

\[
\min\ c_p + c_b + \lambda\,(\mathrm{CVaR}^N + \mathrm{CVaR}^L)
\]

## 三、`duibi.build_kkt_model`

在 **与单层相同的 \(c_p,c_b\)、放置与流守恒、\(u_s,v_s\) 与 \(\zeta_N,\zeta_L\)** 之上，将 **CVaR 极值条件** 用 **KKT 乘子** \(\lambda_{smk},\mu_s\)（算力）与 \(\alpha_{se},\gamma_s\)（链路）及 **指示/互补** 松弛线性化（代码中为 `GenConstrIndicator` 等）。目标仍为

\[
\min\ c_p + c_b + \lambda_{weight}(\mathrm{CVaR}^N + \mathrm{CVaR}^L)
\]

（具体 KKT 等式与互补形式以 `duibi.py` 为准；公式文档给出经济含义即可。）

---

## 四、`duibi.build_epsilon_constraint_model`

**目标**：\(\min\, c_p + c_b\)。

**风险**：与单层相同的 \(\mathrm{CVaR}^N,\mathrm{CVaR}^L\) 表达式，但改为 **硬约束**：

\[
\mathrm{CVaR}^N \le \Gamma_N,\qquad \mathrm{CVaR}^L \le \Gamma_L
\]

---

## 五、`duibi.build_copo_mccormick_model`

**目标**：\(\min\, c_p + c_b\)（不含 \(\lambda\) 乘 CVaR）。

**约束**：同一套节点/链路 CVaR 线性化不等式，另加 **KKT 结构 + McCormick 包络** 近似互补条件（\(\lambda\cdot \mathrm{slack}=0\) 类），使 **CVaR 风险不超过隐式下层最优** 的松弛可解形式。详见 `duibi.py` 中 `lam/alpha` 与 `slack` 约束。

## 六、`cvar_compare.build_teavar_sla_cvar_model`

**决策**：\(y,x^{in},x^{out}\) 同上；场景送达 \(d^{in}_{i,m,p,s},d^{out}_{i,m,q,s}\ge 0\)；SLA CVaR 用 \(\zeta,u_s\)。

### 6.1 成本与期望送达奖励

\[
c_p,\ c_b\ \text{同 duibi};\quad
\mathbb{E}[\mathrm{Del}] = \sum_s \pi_s \sum_{i,m,p} d^{in}_{i,m,p,s} + \sum_s \pi_s \sum_{i,m,q} d^{out}_{i,m,q,s}
\]

（实现中 ingress/egress 均计入。）

### 6.2 SLA「需求未满足」CVaR（与 TEAVAR「1−R/d」同构）

记场景 \(s\) 下任务 \(i\) 的聚合送达为 \(R^{in}_{is}=\sum_{m,p} d^{in}_{i,m,p,s}\)，\(R^{out}_{is}\) 类似。则

\[
u_s\,b^{in}_i \ge b^{in}_i - R^{in}_{is} - b^{in}_i\,\zeta,\qquad
u_s\,b^{out}_i \ge b^{out}_i - R^{out}_{is} - b^{out}_i\,\zeta
\]

\[
\mathrm{CVaR}^{SLA} = \zeta + \frac{1}{1-\beta_{loss}}\sum_s \pi_s u_s
\]

**流与送达耦合**（路径通时）：\(d\le x\)，\(d\le M y\)，\(d\ge x-M(1-y)\)；断路径则 \(d=0\)。**实现中** \(x\) 与需求为 **`<=`** 松弛（非强制拉满管道）。

### 6.3 可选：算力「利用率」CVaR（`lambda_node_cvar>0`）

\[
\mathrm{CVaR}^{nodeUtil} = \zeta_N + \frac{1}{1-\beta_{node}}\sum_s \pi_s \nu_s,\qquad
\nu_s \ge \frac{\sum_i w_{ik} y_{im}}{C^N_{mks}} - \zeta_N\quad \forall s,m,k
\]

### 6.4 可选：算力「未满足量」CVaR（`lambda_compute_sf_cvar>0`）

按资源维定义需求 \(D_{mk}=\sum_i w_{ik} y_{im}\)，超额

\[
e_{mks}=\max(0,\,D_{mk}-C^N_{mks})
\]

（代码用二进制 + Big-M 线性化；\(M\) 取不小于 \(\max_{m,k,s} C^N_{mks}\) 与最大可能 \(D_{mk}\) 之上界以保证 \(D\ll C\) 时可行。）

场景标量损失（与「\(1-A/D\)」同向的聚合近似）：

\[
L_s = \max_{m,k}\ \frac{e_{mks}}{D_{ref}},\qquad
\mathrm{CVaR}^{sf} = \zeta_{sf} + \frac{1}{1-\beta_{sf}}\sum_s \pi_s \phi_s,\quad
\phi_s \ge L_s - \zeta_{sf}
\]

实现中 \(D_{ref}\) 与 Big-M 同量级常数。

### 6.5 总目标

\[
\min\ c_p + c_b + \lambda_{cvar}\mathrm{CVaR}^{SLA}
+ \lambda_{node}\mathrm{CVaR}^{nodeUtil}
+ \lambda_{sf}\mathrm{CVaR}^{sf}
- \omega\,\mathbb{E}[\mathrm{Del}]
\]

（各 \(\lambda\) 为 0 时不建对应项。）

### 6.6 可选：虚拟源/汇接入上界（`add_teavar_virtual_bottleneck_constraints`）

实现中**不增加**图上的显式节点 \(V_s\)，而是在数据对象上携带 **逻辑边可用率** \(\sigma^{vs}_{m,s}\)、\(\sigma^{vt}_{m,s}\in(0,1]\)（字段 `sigma_vs` / `sigma_vt`；缺省 `None` 表示不启用）。在 `del` 与路径 \(\sigma_{es}\) 耦合之后、SLA 的 Rockafellar–Uryasev 行之前，对每个场景 \(s\) 与任务 \(i\) 施加：

\[
R^{in}_{is}=\sum_{m,p} d^{in}_{i,m,p,s}\ \le\ \sum_{m} b^{in}_i\,y_{im}\,\sigma^{vs}_{m,s},
\qquad
R^{out}_{is}=\sum_{m,q} d^{out}_{i,m,q,s}\ \le\ \sum_{m} b^{out}_i\,y_{im}\,\sigma^{vt}_{m,s}.
\]

若未单独给出 \(\sigma^{vt}\)，则与 \(\sigma^{vs}\) 共用同一字典。**含义**：将「经虚拟源/汇进入物理网」建模为与物理送达**串联**的可靠性上界；当任务置于 hub 且 hub↔自身为空路径时，仍可通过 \(\sigma^{vs}_{m,s}<1\) 保留接入缝隙，缓解 SLA CVaR 对 \(\lambda\) 的梯度退化。**数据来源**：`load_b4_joint_data(..., virtual_source=True, virtual_source_sigma=..., virtual_sink_sigma=...)`；`teavar_framework_models` 中 Model **B/C/D** 在自建 MILP 中同样调用该组约束；Model **A** 经 `build_teavar_sla_cvar_model` 已包含。

**与 §6.7 显式 UMCF 的分工**：§6.6 的 `sigma_vs` 方案**不**改 `P_cand`/`E`；§6.7 在 **同一 `data` 对象**上扩展 \(V_s,V_t\) 与单跳虚拟边后，**`build_teavar_sla_cvar_model`、`duibi` physical（Model A–D）与 `teavar_framework_models`（Model A–D）** 的 `xin`/`xout`、链路 `flow_e` 与送达聚合均经 **`teavar_flow_anchors(data)`** 读 `P_cand`（关闭 UMCF 时退化为 hub 径向 \((h,h)\)）。下表第一列专指 §6.6；第二列专指 §6.7 显式图扩展。

| 特性 | §6.6 `sigma_vs` / `sigma_vt` 瓶颈 | §6.7 显式 \(V_s,V_t\)（扩展 `E`/`P_cand`） |
|------|-----------------------------------|---------------------|
| 解决 hub 空路径上 SLA/损失退化 | ✅（串联上界） | ✅（`del` / 径向流走虚拟边 \(\sigma\)） |
| 扩展 `E`/`P_cand` | ❌ | ✅（单跳 \(P_{V_s,m},P_{m,V_t}\)） |
| `duibi` physical 与 `P_cand` 锚点 | 未开 UMCF 时为 hub 径向 | 开 UMCF 时与 `teavar_sla` **同锚点** |
| 多入口「全论文级」编排 | ❌ | 单跳已入优化；多段 K 短路串联待扩展 |

### 6.7 显式虚拟节点 UMCF（**`load_b4_joint_data` / 玩具 `attach_umcf_to_data_object` 开关**）

当 `load_b4_joint_data(..., umcf_virtual_nodes=True)`（或玩具上 `attach_umcf_to_data_object`）时：令 \(n=\texttt{len(M)}\)，\(V_s=n\)、\(V_t=n+1\)；向 `E,B,sigma` 增加边 \((V_s,m)\)、\((m,V_t)\)（容量取现有物理边最大容量），并设 `P_{V_s,m}=[(V_s,m)]`、`P_{m,V_t}=[(m,V_t)]`。**`build_teavar_sla_cvar_model`、`duibi` 四套 physical、`teavar_framework_models` 自建 A–D** 均通过 **`duibi_metrics.teavar_flow_anchors`** 将 `xin`/`del_in` 锚在 \(s_{\mathrm{src}}\to m\)，`xout`/`del_out` 锚在 \(m\to s_{\mathrm{dst}}\)（UMCF 下即 \(V_s,V_t\)）；`add_teavar_virtual_bottleneck_constraints` 在 UMCF 下自动跳过（避免与虚拟边 \(\sigma\) 双重收紧）。**CLI**：`main.py --joint-umcf-teavar`、`duibi.py --umcf-teavar`，可选 `--joint-umcf-sigma` / `--umcf-sigma` 等。

**`main.py --mode joint` 解读注意**：在 **`--joint-stress-zero-s1` 且未开 UMCF** 时，`build_single_layer_model`（physical）与 `build_teavar_sla_cvar_model`（teavar_sla）是 **两套不同 MILP**，可行域不必一致（可出现 physical 不可行而 teavar_sla 仍最优）；公平并排与「应力恢复」叙事以 **同开 `--joint-umcf-teavar`** 或单独跑 **`duibi.py`** 的 physical A–D 为准。详见 README「joint」小节说明。

**可继续扩展**：\(V_s\) 经多段物理路径到 \(m\)、或与 hub 段串联等，以逼近最一般 UMCF 论文叙述。

## 七、`b4_joint_data.load_b4_joint_data`

**语义**：指定 **hub** 节点 \(h\)；任务流量 \(b^{in}_i,b^{out}_i\) 来自 `demand.txt` 矩阵第 \(h\) 行出向 OD 的最大若干条（缩放同 `parsers.read_demand`），再乘以常数 **`demand_scale`**（默认 1，>1 用于整体加压）；\(w,\pi,C^{norm},C^N_{mks},B_e,\sigma\) 来自拓扑与 `node_compute_resources.csv`；\(P_{uv}\) 为图上 \(u\to v\) 至多 \(K\) 条最短简单路径（边序列）。

**场景**：\(s=0\) 全通；\(s=1\) 可对高 `prob_failure` 边置断；\(s=2\) 可对聚合层节点缩小 \(C^N_{mks}\)（与数据文件一致）。

**虚拟接入**：`virtual_source=True` 时为每个物理节点 \(m\) 与场景 \(s\) 写入常数 \(\sigma^{vs}_{m,s}\)（默认 **0.99**）及对称的 \(\sigma^{vt}_{m,s}\)（`virtual_sink_sigma` 缺省则与源相同），供第 6.6 节约束使用。

**UMCF**：`umcf_virtual_nodes=True` 时写 `umcf_vs,umcf_vt` 并扩展 `E,B,sigma,P_cand`（见 §6.7）；**`duibi` physical、`teavar_sla`、`teavar_framework_models` 与 joint 并排求解**共用该图实例。与 `virtual_source` 同时请求时 **以 UMCF 为准**（不写入 `sigma_vs` 字典）。

## 八、`duibi_metrics` 解后指标（非优化，公式）

| 函数 | 公式含义 |
|------|----------|
| `path_up` | 路径上所有边 \(\sigma_{es}>0\)。 |
| `max_link_util_after_solve` | 名义场景 \(s=0\)：\(\max_e \mathrm{flow}_e/(B_e)\)。 |
| `max_node_util_after_solve` | \(s=0\)：\(\max_{m,k} \sum_i w_{ik} y_{im}/C^{norm}_{mk}\)。 |
| `worst_max_link_util_across_scenarios` | \(\max_s \max_e \mathrm{flow}_e/(B_e\sigma_{es})\)。 |
| `worst_max_node_util_across_scenarios` | \(\max_s \max_{m,k} \sum_i w_{ik} y_{im}/C^N_{mks}\)。 |
| `expected_delivery_ratio` | 各任务在放置节点上 \((x^{in}\!/b^{in}+x^{out}\!/b^{out})/2\) 的平均。 |
| `expected_total_delivered_volume` | \(\sum_s \pi_s\sum_{i,m,p} d^{in}_{i,m,p,s}+\cdots\)（路径求和与 `teavar_flow_anchors(data)` 给出的 \((u,v)\) 下 \(P_{uv}\) 一致） |
| `sla_per_scenario_max_demand_loss` | \(L_s=\max_i \max\{1-R^{in}_{is}/b^{in}_i,\,1-R^{out}_{is}/b^{out}_i\}\)。 |

**实现注**：涉及 \(P_{uv}\) 的送达与链路流量指标均以 **`teavar_flow_anchors(data)`** 为 \((u,v)\) 锚点（hub 径向或 UMCF 的 \(V_s,V_t\)）。**放置**、`min_off_hub`、**应力**（切断 hub 出边）仍用物理 **`data.hub`**；二者分工与 README「模型关系」一致。

## 十一、`teavar_framework_models`（TEAVAR 视角与 duibi 四架构对齐）

与 `duibi.py` 中 A–D 平行：**横轴**为 SLA 需求损失 CVaR + 可选算力未满足 CVaR；**纵轴**为加权 / KKT / ε-约束 / McCormick。**流锚点**与 `cvar_compare.build_teavar_sla_cvar_model` 一致：均由 **`teavar_flow_anchors(data)`** 得到 \((s_{\mathrm{src}},s_{\mathrm{dst}})\)，未开 UMCF 时为 hub \(h\) 径向 `P_{hm},P_{mh}`（`h=\texttt{getattr(data,'hub',0)}`），开 UMCF 时为 \(P_{V_s,m},P_{m,V_t}\)。在 `del` 与物理链路耦合之后，**Model B/C/D** 调用 `cvar_compare.add_teavar_virtual_bottleneck_constraints`（当 `data.sigma_vs` 非空且未走 UMCF 图扩展时），与 §6.6 一致；**Model A** 委托 `build_teavar_sla_cvar_model`（支持 `data.umcf_virtual_nodes`，见 §6.7）。**放置**仍用物理 hub \(h\)。

**共通量**：\(c_p,c_b\) 同 `duibi`；期望送达 \(\mathbb{E}[\mathrm{Del}]=\sum_s\pi_s\sum_{i,m,p}d^{in}_{i,m,p,s}+\cdots\)；\(c_{total}=c_p+c_b-\omega\mathbb{E}[\mathrm{Del}]\)。  
SLA 尾部：Rockafellar–Uryasev 标量 \(\zeta,u_s\)，与现有实现一致：\(u_s b^{in}_i\ge b^{in}_i-R^{in}_{is}-b^{in}_i\zeta\)（out 同理），\(\mathrm{CVaR}^{SLA}=\zeta+\frac{1}{1-\beta}\sum_s\pi_s u_s\)。  
算力未满足（可选）：\(D_{mk}=\sum_i w_{ik}y_{im}\)，\(e_{mks}=\max(0,D_{mk}-C^N_{mks})\)，\(L^{sf}_s=\max_{m,k}e_{mks}/D_{ref}\)，\(\phi_s\ge L^{sf}_s-\zeta_{sf}\)，\(\mathrm{CVaR}^{sf}=\zeta_{sf}+\frac{1}{1-\beta_{sf}}\sum_s\pi_s\phi_s\)。

| 模型 | 目标 / 约束 | 实现函数 |
|------|----------------|----------|
| **A** 单层加权 | \(\min\, c_{total}+\lambda_{sla}\mathrm{CVaR}^{SLA}+\lambda_{sf}\mathrm{CVaR}^{sf}\)（\(\lambda_{sf}=0\) 时不建 sf 块） | `build_teavar_model_a` → 委托 `cvar_compare.build_teavar_sla_cvar_model`（`lambda_node=0`） |
| **B** KKT | 与 A 同目标；对 SLA 的每条 \((s,i)\) 探测行加 \(\mathrm{slack}^{SLA}\) 与对偶 \(\alpha\) 的 **Indicator** 互补；若 \(\lambda_{sf}>0\) 且 `kkt_sf=True`，对 \(\phi\) 层 \((s,m,k)\) 再加一套 KKT Indicator | `build_teavar_model_b` |
| **C** ε-约束 | \(\min\, c_{total}\) s.t. \(\mathrm{CVaR}^{SLA}\le\Gamma_{sla}\)，（可选）\(\mathrm{CVaR}^{sf}\le\Gamma_{sf}\) | `build_teavar_model_c` |
| **D** McCormick | \(\min\, c_{total}\)；用线性包络 \(\frac{\alpha}{\alpha_{max}}+\frac{\mathrm{slack}}{\mathrm{slack}_{max}}\le 1\) 松弛 SLA 与 sf 的互补（与 `duibi.build_copo_mccormick_model` 同构思想） | `build_teavar_model_d` |

**说明**：Model D 事后在解上读取的 \(\mathrm{CVaR}^{SLA},\mathrm{CVaR}^{sf}\) 与 Model A 最优 CVaR **不必同界**（松弛松），与 physical 的 Model D 相同 phenomenon。
