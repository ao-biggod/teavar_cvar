# 项目完整画像与分歧点 —— 请给出最终判断

## 一、项目现状（已确认，无争议）

**定位**：故障感知算力–网络联合优化的 MILP。有向 WAN 上联合决定异构算力放置 + 两阶段多路径分流 + **双通道 CVaR**（网络 SLA 未满足 + 算力超额未满足）。Gurobi，B4 为主。

**模型体系**：
- **Model A**（加权 MILP）：min cost + λ·CVaR − ω·E[Del]
- **Model C**（ε-约束）：min cost s.t. CVaR ≤ Γ
- **Model B**（KKT Indicator）：与 A 同解，仅验证
- **Model D**（McCormick）：实测 CVaR 偏 210%，放弃
- **Physical 对照**：利用率 CVaR（不同风险语义）
- **Model M/M-C**：货币化 CVaR（量纲统一，κ 对应违约金）

**数据**：B4 是唯一 gold standard（异质容量 + 完整 demand + 手工拉宽价格 8×），ATT/IBM/Abilene/Sprint 可做泛化，XNet 缺 prob_failure 列，Nextgen/Custom/Custom2 用合成 demand。

**代码**：main.py 支持 joint/monetary/teavar 三种模式，Model C 已进管线，YAML 文件存在但代码不读（硬编码），stress 有两种模式（top-k 断链 vs hub 全出边切断）。

## 二、核心诊断：实验装置无法展示卖点（已达成共识）

当前三组实验：
- hub+stress：Physical 全不可行，teavar SLA=1.0 饱和
- 无 stress：teavar SLA=0 退化
- UMCF+stress：teavar SLA=0 退化

**旗舰图应是一张"固定成本下 CVaR^SLA × CVaR^sf 的二维权衡前沿"**，展示降低送达风险如何以抬高算力风险为代价。但目前 CVaR^SLA 要么 0 要么 1，CVaR^sf 恒为 0，λ 扫描全失效。根因：二元切断 + 容量有余量 = 没有竞争区间。

**修复方向**：部分 σ（连续可用率而非 0/1）+ 容量按故障缺口标定 + 加任务数触发节点溢出。
