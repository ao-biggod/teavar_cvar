 这个 Big-M 线性化的目标是把非线性的 $\max(0, \cdot)$ 函数用纯线性约束表达出来，使得 MILP 求解器能处理。

  ---
  问题：为什么要线性化

  MILP 求解器只接受线性约束。$e = \max(0, D-C)$ 是一个分段函数：

  如果 D > C  →  e = D - C
  如果 D ≤ C  →  e = 0

  它不是一条直线，不能直接写进求解器。需要用 Big-M + 二元变量来"选段"。

  ---
  约束逐条拆解

  已有的两条（下界）

  $$e \ge D - C$$
  $$e \ge 0$$

  这两条联合起来就是 $e \ge \max(0, D-C)$。但光有下界不够——求解器会让 $e$ 取任意大的值（只要满足 $\ge$），需要上界把 $e$
   压回到恰好等于 $\max(0, D-C)$。

  新增的 Big-M 上界（关键）

  引入一个二元开关 $w \in {0, 1}$，含义是"是否超额"：

  $$w = 1 \text{ 表示 } D > C \text{（有缺口）}$$
  $$w = 0 \text{ 表示 } D \le C \text{（没缺口）}$$

  然后用 $M$（一个足够大的常数）来"激活/关闭"上界：

  约束 A： $e \le (D - C) + M \cdot (1 - w)$

  ┌───────┬───────────────────┬──────────────────────────────────────────────────────┐
  │  $w$  │     约束变成      │                         含义                         │
  ├───────┼───────────────────┼──────────────────────────────────────────────────────┤
  │ $w=1$ │ $e \le D - C$     │ $e$ 被卡在 $D-C$，配合下界 $e \ge D-C$，得 $e = D-C$ │
  ├───────┼───────────────────┼──────────────────────────────────────────────────────┤
  │ $w=0$ │ $e \le (D-C) + M$ │ $M$ 足够大，这条约束形同虚设，不起作用               │
  └───────┴───────────────────┴──────────────────────────────────────────────────────┘

  约束 B： $e \le M \cdot w$

  ┌───────┬───────────┬────────────────────────────────┐
  │  $w$  │ 约束变成  │              含义              │
  ├───────┼───────────┼────────────────────────────────┤
  │ $w=1$ │ $e \le M$ │ $M$ 很大，不起作用             │
  ├───────┼───────────┼────────────────────────────────┤
  │ $w=0$ │ $e \le 0$ │ 配合下界 $e \ge 0$，得 $e = 0$ │
  └───────┴───────────┴────────────────────────────────┘

  ---
  图示

  D > C 的情况（有缺口）：
    ─────────────────────────────────────
    下界:  e ≥ D-C,  e ≥ 0
    上界A: e ≤ D-C       (w=1, M项清零)
    上界B: e ≤ M         (w=1, 不起作用)
    → e = D-C  ✓

  D ≤ C 的情况（没缺口）：
    ─────────────────────────────────────
    下界:  e ≥ D-C(负数),  e ≥ 0  →  实际下界是 0
    上界A: e ≤ (D-C)+M   (w=0, M项保留, 不起作用)
    上界B: e ≤ 0         (w=0, M项清零)
    → e = 0  ✓

  ---
  为什么叫 Big-M

  $M$ 是一个足够大的常数（通常取问题中变量可能的最大值的数倍）。它必须满足：

  - 当 $w=0$ 时，$M$ 要大到让 $(D-C) + M$ 绝对大于 $e$ 的任何可能取值
  - 当 $w=1$ 时，$M$ 要大到让 $M$ 绝对大于 $D-C$

  $M$ 如果取得太小，会把合法解错误地排除（truncation）；取得太大，会造成数值不稳定、求解速度下降。实际工程中 $M$ 取 $D$
  的可能最大值（比如 $\sum_i \max_k w_{ik}$）加上一个安全裕度。

  ---
  整体逻辑

  四条约束联合，等价于：

  min  e
  s.t. e ≥ D-C, e ≥ 0, e ≤ (D-C)+M(1-w), e ≤ Mw, w∈{0,1}

  因为目标函数中 $e$ 总是带正系数出现（成本/惩罚项），求解器会自动把 $e$ 往小压，不会出现 $e$ 比 $\max(0, D-C)$
  大的情况。所以下界约束 $e \ge D-C$ 和 $e \ge 0$ 中的至少一条会在最优解处取等号，加上 Big-M 上界精确卡位，最终 $e =
  \max(0, D-C)$ 成立。

  一句话：用二元变量 $w$ 做开关，$M$ 做"无限大"，把分段函数的两个分支分别激活/屏蔽，实现对 $\max$ 的等价线性表达。
