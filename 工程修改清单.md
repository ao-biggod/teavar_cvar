## 五、已知实验陷阱（与最终路线图 C-1~C-5 一致）

> ⚠️ **适用范围**：以下 1–9 条均来自 **hub 径向**（`routing_mode=hub`）下的 B4 实验观察。**per-task OD 下 SLA/sf 退化形态会不同**（例如不同 src/dst 天然分散放置，「全堆 hub」类退化可能减弱或消失）——**阶段 1 验收后须重新跑 P0 并更新本节**。

1. **hub-stress 进主表** → SLA=1.0 饱和、λ 无效、Physical 不可行 → **仅 Physical foil**（C-2）  
2. **UMCF 降 σ** → SLA 仍 0 → **主表不用 UMCF**（C-3）  
3. **virtual-source σ=0.90** → SLA≈0.10 但 λ–SLA 仍平 → **主机制用链路部分 σ+η**（决策 #11）  
4. **\|I\|=8** → computeSf 常 0 → P0 默认 **\|I\|=12**（§5.1）  
5. **nodeUtil CVaR ~0.92** → 主实验 **λ_n=0**（C-1）  
6. **ω·E[Del] 很大** → 可能淹没 λ → **ω=0 消融**（C-4）  
7. **demand_scale=25** → P0 后默认 **η 标定**；显式 `--joint-demand-scale` **覆盖** η（消融用）  
8. **HBM w=32 vs edge 容量** → 放置资格特征；加载需 valid_assign 检查（C-5）  
9. **Γ 网格** → 须 **V-1/V-2 通过后** 再用 Model A 标定 Γ；否则鸡-蛋退化（§5.2 前置条件）
