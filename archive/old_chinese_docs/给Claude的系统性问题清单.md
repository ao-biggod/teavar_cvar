## 二、模型体系（我需要你理清的东西）

### 当前有四套模型 + 两套扩展：

**SLA 主线（双 CVaR）：**
- **Model A**：min cost + λ_sla·CVaR^SLA + λ_sf·CVaR^sf − ω·E[Del]。单层加权 MILP，精确。
- **Model C**：min cost s.t. CVaR^SLA ≤ Γ_sla, CVaR^sf ≤ Γ_sf。ε-约束，精确。
- **Model B**：同 A + KKT Indicator 互补约束。仅验证用（慢、二元变量多）。
- **Model D**：McCormick 松弛。实测 CVaR 偏差 210%，不可作主求解器。

**对照/扩展：**
- **Physical A/C**：利用率 CVaR（算力拥塞 + 链路拥塞），而非 SLA 需求未满足。用于对照。
- **Model M / M-C**：纯货币化 CVaR。场景账单 L_s = c_p + c_b(s) + κ·Shortfall，目标 min CVaR(L) 或 min E[L] s.t. CVaR≤Γ。量纲统一，κ 直接对应 SLA 违约金。

### 我的具体问题：

1. **Model A 和 C 到底是什么关系？** 我的理解是：A 给运营商画 Pareto 地图（扫 λ），C 给客户报价（给定 Γ 求最低成本）。但当前实验里 A 和 C 在同等 Γ 下给出相同解——那 C 的价值到底在哪？什么情况下 C 会给出 A 给不了的信息？

2. **Model B 和 D 的去留。** D 我打算正文一句带过（"McCormick 在此场景偏差 210%"）+ 附录小表。B 是否需要跑一组实验证 gap=0%，还是理论说明就够？

3. **Physical 对照应该放正文还是附录？** 它是不同风险语义（利用率 vs 需求未满足），放在主表里会不会分散注意力？

4. **Model M/M-C 应该放正文还是附录？** 它解决了 Model A 的"λ 无物理含义"问题，但结构上多了一套定义。是作为"Discussion：货币化重构"好，还是压到附录？

5. **最优论文模型配置是什么？** 即"正文讲哪些、附录放哪些、完全不提哪些"。
