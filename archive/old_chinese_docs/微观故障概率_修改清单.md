## 二、场景生成算法（M1 详细设计）

### 函数签名

```python
def _generate_link_failure_scenarios(
    prob_edges: list[tuple[int, int, float]],   # (src, dst, prob_failure)
    *,
    prune_threshold: float = 1e-5,               # 概率低于此值的场景被丢弃
    max_simultaneous_failures: int = 2,           # 最多同时断几条边
    scenario_sigma: float = 0.0,                  # 故障边的可用率（0=硬断，0.80=部分降级）
) -> tuple[dict[int, float], dict[int, dict[tuple[int,int], float]], dict]:
    """
    返回:
      scenario_probs: {s_id: probability}       — 归一化后的场景概率
      scenario_sigmas: {s_id: {edge: sigma}}     — 每场景的边可用率（仅存非 1.0 的边，节省内存）
      metadata: {total_scenarios, pruned_count, threshold, ...}
    """
```

### 生成逻辑

```
1. all-up 场景（s=0）：
   π_0 = ∏_{e} (1 - pf[e])
   所有边 σ = 1.0

2. 单边故障场景（s=1,2,...,38）：
   对每条边 e_i：
     π_i = pf[e_i] × ∏_{j≠i} (1 - pf[e_j])
     σ[e_i] = scenario_sigma
     其他边 σ = 1.0
   若 π_i < prune_threshold → 丢弃

3. 双边故障场景（s=39,40,...）：
   对每对边 (e_i, e_j), i<j：
     π_ij = pf[e_i] × pf[e_j] × ∏_{k≠i,j} (1 - pf[e_k])
     σ[e_i] = σ[e_j] = scenario_sigma
     其他边 σ = 1.0
   若 π_ij < prune_threshold → 丢弃

4. 归一化：
   将所有保留场景的概率缩放到 Σπ = 1.0
   记录被丢弃的概率总量（用于论文说明剪枝损失）
```

### B4 估算（pf ∈ [0.00184, 0.00450]，prune_threshold=1e-5）

| 故障数 | 场景数 | 单场景概率范围 | 保留？ |
|--------|--------|-------------|--------|
| 0 | 1 | ~0.89 | ✅ |
| 1 | 38 | ~0.002–0.004 | ✅ 全部 |
| 2 | 703 | ~4e-6–2e-5 | 🟡 部分保留（概率较高的双边故障对） |
| 3+ | ~8000+ | <1e-7 | ❌ 全部丢弃 |

**预期总场景数：~40–80**（1 all-up + 38 single + ~10–40 double）

## 三、s=2 算力降额如何接入

两种选择：

### 方案 A：与链路故障叠加（推荐）

对算力降额 s=2（原 π=0.1），不再作为独立场景。而是**对每个保留的链路故障场景，以 10% 概率叠加算力降额**：

```
对每个保留的链路故障场景 s：
  π[s_normal] = π[s] × 0.9
  π[s_degraded] = π[s] × 0.1
  s_degraded 下 aggregation 节点容量 × s2_derate（0.40）
```

效果：场景数翻倍（~80–160），但逻辑简单——**链路问题和算力问题可以同时发生**。

### 方案 B：保留为独立场景

算力降额不出现在链路故障场景里，而是额外加 1 个独立场景（s=n+1, π≈0.1）。**链路易处理**，但链路故障和算力降额不能同时发生。

**建议方案 A**，论文可以写"模型支持链路与算力联合故障的任意组合，实验中算力降额与链路失效以独立概率叠加"。

## 四、CLI 参数（M4 详细设计）

| 新参数 | 默认值 | 说明 |
|--------|--------|------|
| `--scenario-prune-threshold` | `1e-5` | 场景概率剪枝阈值 |
| `--scenario-max-failures` | `2` | 枚举的边故障数上限 |
| `--scenario-sigma` | `0.0` | 故障边的可用率（0=硬断，0.80=部分降级，复用 P0 的 0.80） |
| `--scenario-compute-degrade-mode` | `overlay` | `overlay`=叠加（方案 A），`standalone`=独立场景（方案 B） |

**废弃的参数**（与 top-k/uniform σ 耦合）：
- `--scenario-s1-link-sigma` → 被 `--scenario-sigma` 替代
- `--scenario-s1-link-k` → 被场景生成替代（不再有"top-k"概念）
- `--joint-stress-zero-s1` → 被场景生成替代（Physical foil 单独处理）
