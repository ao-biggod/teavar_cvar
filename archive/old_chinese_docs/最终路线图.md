## 五、实验装置：η-标定原理

选定最严重场景 **s\*** 的链路可用率 σ_{es\*}，计算瓶颈幸存容量 **C_surv(s\*)**，设过载比 **η**，使总需求 Σb_i = η·C_surv(s\*)。η>1 保证最坏态出现结构性缺口；**ρ ≈ (1 − 1/η)×100%** 从拓扑涌现，不是手调魔数。

### 5.1 默认实验配置（P0 操作规格）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **s\*** | **s = 1** | 链路降级场景（s=0 名义，s=2 算力降额） |
| **s1 机制** | top-k 高 prob 边设 **σ = 0.80**（非 0/1） | k = **4**（`scenario_s1_link_k`） |
| **η** | **1.3** | 总需求 = η·C_surv(s*)；涌现 ρ ≈ 23% |
| **\|I\|** | **12** | 8 任务上 sf 恒 0 已验证 |
| **min_off_hub** | **2** | 反退化：避免全堆 hub |
| **s2_derate** | **0.40**（P0 实验；CLI 覆盖） | 代码默认 **0.60**（E-09）；P0 用 **0.40** 在 \|I\|=12 下加大算力溢出压力；V-2 仍失败则 **回退 0.60** |
| **λ_n** | **0** | C-1 |
| **λ_sf** | **1.0** | 双 CVaR 主实验 |

| **k_paths** | **4** | hub↔节点候选路径 |
| **主 SLA 机制** | **链路部分 σ + η** | virtual-source σ≈0.90 **仅消融** |
| **hub 全断** | **不开** | 仅 Physical foil 脚本使用 |

**C_surv(s*) 操作定义（实现目标）：**

1. 在 s=1 下，对 hub→各节点（或 s→t 锚点）候选 ingress 路径，取路径瓶颈容量 **min_{e∈p} σ_e·B_e**；
2. **C_surv(s*) = max_p 上述瓶颈**（或 hub 割集上最小幸存流，取实现中较保守者）；
3. 由 η **反算** `demand_scale` / 各任务 b_in,b_out，**取代** 手调 `demand_scale=25`。

**代码落点（P0-4）：** `b4_joint_data.py` — `estimate_surv_capacity()` + `apply_eta_demand_calibration(η)` + 部分 σ 赋值。

**`demand_scale` / `--joint-demand-scale` 去留：**

- CLI **保留**（向后兼容、消融用），但 **默认行为改为 η 标定自动反算 scale**；
- 未显式指定 `--joint-demand-scale`（或等价 `--eta`）时，loader 按 η 与 C_surv(s*) 计算 demand；
- **手动** `--joint-demand-scale <值>` 时 **覆盖 η 标定**（用于与旧 scale=25 或 η 敏感性对比）；
- `main.py` / `run_gamma_frontier.py` 推荐新增 **`--eta`**（默认 1.3），与 `--joint-demand-scale` 二选一优先：scale 显式 > η 标定 > 代码兜底。

## 六、论文的最强防御线（三句话）

1. **"为什么 path-based"** → 与 TEAVAR 同框架直接可比；MPLS 运维现实；flow-based 是 scaling（Future Work）。
2. **"为什么连续 σ"** → 二元中断是连续降级特例；我们**推广** TEAVAR 故障模型。
3. **"λ 怎么选"** → C 用 Γ（合同语言）；M 半节说明 κ 统一量纲（附录完整）。
